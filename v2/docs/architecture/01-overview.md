# EduQA V2 架构设计文档

## 1. 架构愿景

EduQA V2 是一个企业级的教育问答系统，采用**领域驱动设计 (DDD)** 和**六边形架构**思想，实现高内聚、低耦合、可测试、可扩展的系统架构。

## 2. 核心架构原则

### 2.1 分层架构
```
┌─────────────────────────────────────────────────────────────┐
│                    接口适配层 (Interfaces)                    │
│           HTTP API / WebSocket / CLI / Admin                │
├─────────────────────────────────────────────────────────────┤
│                    应用层 (Application)                     │
│     应用服务 / DTO / 事务控制 / 用例编排                      │
├─────────────────────────────────────────────────────────────┤
│                    领域层 (Domain)                          │
│     领域模型 / 领域服务 / 领域事件 / 业务规则                 │
├─────────────────────────────────────────────────────────────┤
│                  基础设施层 (Infrastructure)                │
│     数据库 / 缓存 / LLM / 向量服务 / 文件存储                 │
├─────────────────────────────────────────────────────────────┤
│                    核心层 (Core)                            │
│     配置管理 / 日志 / 异常 / 事件总线 / DI容器                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 依赖方向
**内层不依赖外层，外层依赖内层**
- Domain 层：最纯净，不依赖任何框架
- Application 层：依赖 Domain
- Infrastructure 层：依赖 Domain 和 Application
- Interfaces 层：依赖所有内层

### 2.3 关键技术选型
| 层次 | 技术 | 说明 |
|------|------|------|
| Web框架 | FastAPI | 高性能异步框架 |
| ORM | SQLModel + SQLAlchemy 2.0 | 类型安全ORM |
| 向量库 | pgvector | PostgreSQL扩展 |
| 缓存 | Redis + 本地LRU | 多级缓存 |
| 配置 | Pydantic Settings | 环境分离 |
| 日志 | structlog | 结构化日志 |
| DI | dependency-injector | 依赖注入容器 |
| 前端 | Vue3 + TypeScript + Pinia | 企业级前端 |

## 3. 领域划分

### 3.1 核心领域
1. **Expert (专家领域)** - 学科专家管理、路由策略
2. **Knowledge (知识领域)** - 知识库管理、向量检索
3. **Session (会话领域)** - 问答会话、历史管理
4. **Experiment (实验领域)** - 基准测试、实验控制
5. **Iteration (迭代领域)** - 自我迭代、知识进化
6. **Training (训练领域)** - 微调数据、训练任务

### 3.2 领域边界
```
ExpertDomain
├── Expert (聚合根)
├── ExpertPool (领域服务)
└── RoutingStrategy (值对象)

KnowledgeDomain
├── KnowledgeItem (聚合根)
├── Tier0Knowledge (实体)
├── RetrievalService (领域服务)
└── KnowledgeEvent (领域事件)

SessionDomain
├── Session (聚合根)
├── Message (实体)
├── QAInteraction (值对象)
└── QualityAssessment (值对象)
```

## 4. 数据流设计

### 4.1 问答流程
```
用户提问
    │
    ▼
[专家路由] ──► VL模型识别学科
    │
    ▼
[RAG检索] ──► 向量检索相关知识
    │
    ▼
[LLM生成] ──► 专家Prompt + RAG上下文
    │
    ▼
[质量检查] ──► 云端质检
    │
    ▼
[知识入库] ──► 错题→Tier0→去重→向量化
    │
    ▼
返回回答
```

### 4.2 实验流程
```
创建实验配置
    │
    ▼
[实验编排器] ──► 按顺序执行6组实验
    │
    ▼
[基准测试] ──► 50题/组，种子固定
    │
    ▼
[结果收集] ──► 准确率、响应时间
    │
    ▼
[知识迭代] ──► 错题自动入库
    │
    ▼
[下一轮实验] ──► 使用新知识库
    │
    ▼
生成对比报告
```

## 5. 非功能性需求

### 5.1 性能目标
- 单次问答响应 < 3s (含LLM)
- RAG检索 < 100ms
- 向量相似度计算 < 50ms
- 支持并发 100 QPS

### 5.2 可靠性
- 数据库连接池自动管理
- LLM调用超时和重试
- 服务降级策略
- 优雅关闭

### 5.3 可观测性
- 结构化日志 (JSON)
- 分布式追踪
- 关键指标监控
- 健康检查端点
