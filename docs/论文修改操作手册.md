### 3.2 补充缓存架构说明（3.3.2节）

**在"专家动态生命周期管理"后添加**：
```
3.3.4 四级缓存架构设计

为降低响应延迟，系统实现四级缓存架构：
- L1（内存缓存）：进程级缓存，存储专家实例和向量结果
- L2（Redis缓存）：分布式缓存，支持跨进程共享（可选配置）
- L3（PostgreSQL）：持久化存储
- 冷启动：按需加载专家模型

当Redis未配置时，系统自动降级为L1内存缓存，保证可用性。
```

### 3.3 补充多模态输入说明（3.4.1节）

**修改内容**：
```
原文：
支持图文混合输入

改为：
支持三种输入模式：
(1) 纯文字输入：传统问答模式
(2) 图文混合输入：用户上传图片并附带文字说明
(3) 纯图片输入：用户仅上传题目图片，系统自动使用
    "请解答图片中的题目"作为默认提示
```

## 第六步：更新参考文献格式

### 6.1 检查引用格式一致性

**要求**：
- 期刊论文：作者. 标题[J]. 期刊名, 年, 卷(期): 页码.
- 会议论文：作者. 标题[C]//会议名. 地点: 出版社, 年: 页码.
- arXiv：作者. 标题[J]. arXiv preprint arXiv:编号, 年.

**示例修改**：
```
不规范：
[01] Vaswani A et al. Attention is all you need. NIPS 2017.

规范：
[01] Vaswani A, Shazeer N, Parmar N, et al. All that is needed is attention[C]//Advances in Neural Information Processing Systems. Long Beach, CA: Curran Associates, 2017: 5998-6008.
```

## 修改顺序建议

**推荐修改顺序**：

1. **先删后改**：先删除不需要的章节（第1步）
2. **技术参数**：更新权重、阈值等硬数据（第3步）
3. **章节内容**：修改各章节描述（第2步）
4. **实验数据**：运行系统获取真实数据（第5步）
5. **图表更新**：替换截图、重绘架构图（第4步）
6. **最终检查**：使用检查清单（第7步）

**预计工作量**：
- 删除内容：30分钟
- 修改描述：2小时
- 运行实验获取数据：1-2小时（视测试集大小）
- 更新图表：1小时
- 最终检查：30分钟

**总计：约5-6小时**
