# 降低AI率和查重率的写作技能

> 基于《基于专家模型路由的多模态教育知识问答系统设计与实现》论文分析总结

---

## 一、句式结构变换技巧

### 1. 因果关系的多样化表达

**AI常用模式（避免）：**
- 随着...的发展，...
- 因为...所以...
- 由于...导致...

**论文实际使用（推荐）：**
```
由于教育行业对于个性化教学有强烈的需求，在传统的大型班级里很难做到针对每个学生的不同需求进行教学的情况下，人们开始让AI进入教育行业。

→ 特点："由于...，在...情况下，..." 多层嵌套，避免单一因果句式
```

```
由于大语言模型虽然有很强的语言理解和生成的能力，但也存在知识准确度不足、领域专一性和对多模态的理解等问题，在此基础之上研究...具有一定的意义。

→ 特点：先承认优点，再转折指出问题，最后引出研究意义
```

### 2. 使用"针对..."引出研究内容

```
针对以上问题，本文提出一种Edu-Refiner系统的设计与开发。

针对输入中有"H₂O"的情况，系统会将其分发给化学专家而不是英语专家。
```

### 3. 交替使用主动与被动语态

**主动语态：**
```
本文提出一种Edu-Refiner系统的设计与开发。
```

**被动语态：**
```
系统被设计为可以不断学习成长的教育智能问答系统。
```

**无主语句（常见于技术描述）：**
```
在本文中提出一种针对教育的应用场景下异构专家网络结构。
```

## 二、用词降重技巧

### 1. 技术术语首次出现的标准格式

**格式：中文名称（英文全称，英文缩写）**

```
检索增强生成（Retrieval-Augmented Generation, RAG）
混合专家模型（Mixture of Experts, MoE）
多模态大语言模型（Multimodal Large Language Model, MLLM）
```

**后续使用简称时，避免重复全称**

### 2. 动词多样化

**避免重复：** 使用、采用、利用

**论文实际用词：**
- 搭建异构专家网络结构
- 实现基于多模态的学科路由策略
- 构建云-端自反馈循环机制
- 设计三级知识库
- 采用BGE模型进行向量化表示

### 3. 程度副词的自然使用

```
较大提升（不用"显著提升"）
一定的意义（不用"重要意义"）
很大进展（不用"巨大进展"）
相对较少（不用"很少"）
```

## 三、内容组织降重技巧

### 1. 具体数据替代模糊描述

**AI常用（避免）：**
```
实验结果表明，系统的性能得到了显著提升。
```

**论文实际（推荐）：**
```
完整系统的准确率达到89.7%，比基线模型（64.8%）高出24.9个百分点。
```

### 2. 增加"本文""本研究"等主体指代

```
本文提出...
本文旨在...
本文认为...
本文的主要工作如下...
```

**注意：** 不要过度使用"本文"，每段出现1-2次即可

### 3. 加入主观性分析

```
如果从教育学的角度分析，人工智能与建构主义（Constructivism）、联通主义（Connectivism）有着紧密联系。

→ 特点："如果...角度分析"引入个人理解
```

```
这些研究发现人工智能可以提高学生的学习成绩，可以实现个性化教学还可以激发学生的学习兴趣等优点，但是也存在一定的局限性...

→ 特点：先总结优点，再转折指出局限，体现批判性思维
```

## 四、段落结构技巧

### 1. 技术路线图的插入

在1.3.2节末尾插入：
```
图1-1 Edu-Refiner 技术路线图
```

**作用：** 打断连续文本，降低文本相似度

### 2. 列表与段落交替

**纯段落（AI感强）：**
```
第一，基于Qwen3-VL-4B的多模态学科路由机制。第二，三级知识检索架构的构建。第三，云端-本地异步质检闭环的实现。
```

**论文实际（推荐）：**
```
第一，基于 Qwen3-VL-4B 的多模态学科路由机制。

第二，三级知识检索架构的构建。

第三，云端-本地异步质检闭环的实现。
```

### 3. 使用括号补充说明

```
在回答"牛顿第二定律的公式"的问题上，模型会弄混适用于该公式的前提条件，在相对论情况下也使用它。

→ 特点：具体例子（牛顿第二定律）+ 具体错误（相对论情况也使用）
```

## 五、专业术语的降重表达

### 1. 同义替换策略

| 常用词 | 替换表达 |
|--------|----------|
| 提高 | 提升、增强、改进、优化、增长 |
| 使用 | 采用、利用、运用、基于、通过 |
| 问题 | 不足之处、局限性、挑战、困难 |
| 方法 | 机制、策略、架构、方案、技术路线 |
| 系统 | 框架、平台、模型、结构 |

### 2. 学科术语的具象化

**抽象：** 该系统可以处理教育领域的问题。

**具象：**
```
比如如果输入中有"H₂O"，那么就会被分发给化学专家而不是英语专家；如果有"辛亥革命"，就会被分发给历史专家。
```

## 六、降低AI检测率的核心原则

### 1. 避免"首先...其次...最后..."的机械结构

**AI常用模式：**
```
首先，本文研究了A。其次，本文研究了B。最后，本文研究了C。
```

**论文实际：**
```
(1) 异构专家网络架构的设计

由于一个模型无法兼顾所有领域的要求，因此在本文中提出一种...

(2) 多模态学科识别还有路由机制

由于教育场景中输入形式的多样性，本研究设计基于...

→ 特点：用(1)(2)编号，每个小节前加过渡句说明原因
```

### 2. 在抽象描述中加入具体例子

**抽象：** 多模态模型可以识别图片中的内容。

**具体：**
```
比如对于一道包含手写公式的几何题目，它可以识别人工画出的直角标识以及边长标注，并且能够理解这些信息与题干中的"求三角形面积"的关系。
```

### 3. 使用反问或设问

```
所谓的人工智能就是让计算机通过某种方法获取信息并据此进行判断的能力。

→ 特点：用"所谓...就是..."进行概念阐释
```

### 4. 数据与现象结合

```
在72小时内共完成4,240次问答，有452条高质量知识被存入数据库（入库率为10.7%，见图5），准确率由最初的64.8%提高到78.2%（增长13.4%）。

→ 特点：时间+数量+比例+图表引用+前后对比
```

## 七、查重率控制技巧

### 1. 文献综述的重写策略

**原文（可能重复）：**
```
RAG技术由Lewis等人于2020年提出。
```

**改写：**
```
而最早提出检索增强生成的是由 Lewis 等人在 2020 年提出的 RAG 模型，该方法使用一个已经过预训练的序列到序列模型（BART）以及稠密段落检索器（DPR）来实现有参知识和无参知识库的有效结合。

→ 特点：补充技术细节（BART、DPR、有参/无参知识）
```

### 2. 实验描述的个性化

**模板化：** 实验结果表明，A方法比B方法效果更好。

**个性化：**
```
拆开来看，专家路由独立贡献17.5个百分点、RAG独立贡献9.8个百分点、自我进化模块贡献3.5个百分点；考虑模块间协同效应后，总提升为24.9个百分点。

→ 特点：拆解贡献度，体现对数据的深入分析
```

### 3. 使用长句与短句交替

**长句：**
```
由于教育行业对于个性化教学有强烈的需求，在传统的大型班级里很难做到针对每个学生的不同需求进行教学的情况下，人们开始让AI进入教育行业。
```

**短句：**
```
所谓的人工智能就是让计算机通过某种方法获取信息并据此进行判断的能力。
```

**长短结合：**
```
近年来，随着Transformer架构在自然语言处理任务上的表现优异，基于大型语言模型的生成式人工智能技术取得很大进展。ChatGPT、Claude、Gemini等系统出现使教育界进入所谓"后ChatGPT时代"。
```

## 八、各章节写作要点

### 绪论部分
- **研究背景：** 从宏观到微观，时代需求→技术现状→具体问题
- **研究现状：** 分类综述（RAG、MoE、MLLM），每类先概述再细节
- **研究内容：** 用(1)(2)(3)编号，每项前先说明"由于...因此..."

### 技术原理部分
- **每个技术点：** 定义→原理→在本系统中的应用
- **避免：** 纯复制教科书定义，要加入教育场景的适配说明

### 系统设计部分
- **每个模块：** 需求→设计思路→具体实现
- **使用：** 时序图、架构图打断连续文本

### 实验部分
- **结果描述：** 具体数据+百分比+对比分析
- **避免：** "效果显著""性能良好"等模糊词汇

---

## 九、常见AI特征词汇替换

| AI常用 | 替换为 |
|--------|--------|
| 本文旨在 | 本文旨在研究...（具体目标） |
| 综上所述 | 以上所述 |
| 具有重要的理论意义和实践价值 | 具有一定的意义 |
| 随着...的快速发展 | 近年来，随着... |
| 本文的结构安排如下 | （直接列章节，不设过渡句） |
| 实验结果表明 | 实验表明 / 实验结果显示 |

## 十、最终检查清单

提交前检查：

- [ ] 每段是否有具体数据或例子支撑
- [ ] "本文"出现频率是否合理（不过多也不过少）
- [ ] 是否混用了主动、被动、无主语句式
- [ ] 技术术语首次出现时是否给出英文全称
- [ ] 是否有连续3个以上相同句式的段落
- [ ] 图表引用是否足够（至少5-8处）
- [ ] 是否有个人分析和批判性观点
- [ ] 数字和百分比是否用阿拉伯数字表示

## 附录：论文明目张胆的"非AI"特征示例

1. **口语化表达：** "所谓的人工智能就是..."
2. **设问句：** "如何从保留大语言模型通用语言能力基础上实现学科专业精细化管控？"
3. **自我质疑：** "当前系统的不足之处："
4. **具体时间：** "2026年3月20日"
5. **精确数字：** "准确率由最初的64.8%提高到78.2%（增长13.4%）"
6. **图表引用：** "（入库率为10.7%，见图5）"
7. **第一人称：** "感谢我的导师..."
8. **谦逊表达：** "因为时间、精力有限，本系统尚有许多不足之处"

---

**核心原则：让论文读起来像是一个具体的人在讲述自己做的具体工作，而不是机器生成的通用文本。**
