# 🎓 EduQA System - VIBECODING PRD 文档

> **VIBECODING**: 氛围驱动开发 (Vibe-Driven Development)  
> 以用户体验和系统氛围为核心，构建具有灵魂的教育问答系统

---

## 📋 项目愿景 (Vision)

### 核心使命
构建一个**有温度、会进化、懂学科**的智能教育助手，让每个学生都能获得个性化的专业指导。

### 氛围关键词
`温暖` `专业` `进化` `陪伴` `成长`

---

## 🎯 产品定义

### 一句话描述
基于专家模型路由的多模态教育知识问答系统，具备自我迭代进化能力。

### 核心价值主张
1. **精准学科识别** - 问题该找谁，系统知道
2. **动态知识积累** - 越用越聪明，持续进化
3. **多模态交互** - 文字、图片都能问
4. **实验级精度** - 支持严谨的学术研究对比

## 🎭 用户角色 (Personas)

### 1. 学生小明 (核心用户)
- **画像**: 高中生，理科薄弱，晚上10点还在刷题
- **痛点**: 搜题软件只有答案没有讲解，问老师太晚了不好意思
- **场景**: 拍一道数学题，系统识别题型，数学专家给出详细推导
- **情感需求**: 被理解、被鼓励、不觉得自己笨

### 2. 研究员张老师 (论文用户)
- **画像**: 教育技术方向研究生，正在做对比实验
- **痛点**: 缺乏完整的实验基线系统，数据采集困难
- **场景**: 一键切换实验模式，批量跑基准测试，导出论文图表
- **情感需求**: 数据可信、实验可控、结果可复现

### 3. 系统管理员 (运维用户)
- **画像**: 学校IT部门，负责系统稳定运行
- **痛点**: 系统出问题不知道哪里坏了，监控不到位
- **场景**: 看大盘监控、收告警、扩容服务
- **情感需求**: 一切尽在掌控、问题早发现早解决

## 🚀 核心功能模块

### 模块1: 学科识别路由器 (Vibe: 精准)
```
用户输入 → VL模型分析 → 学科识别 → 专家匹配 → 路由分发

氛围要点:
- 识别过程可视化（显示识别置信度）
- 多学科问题时给出置信度分布
- 识别错误时提供手动切换入口
```

### 模块2: 专家知识库 (Vibe: 专业)
```
学科专家 = 角色设定 + 知识库 + 微调权重

氛围要点:
- 每个专家有独特的"性格"（数学严谨、语文诗意）
- 知识来源透明展示（从哪本书/哪个数据集来的）
- 专家能力可视化（已学多少知识、准确率统计）
```

### 模块3: RAG检索引擎 (Vibe: 广博)
```
多级检索策略:
├─ T1: 学科库（精准匹配）
├─ T2: 通用库（广度补充）
└─ T3: 本地知识库（进化成果）

氛围要点:
- 检索过程可见（显示检索路径）
- 结果置信度标注
- 引用来源可追溯
```

### 模块4: 自我迭代飞轮 (Vibe: 进化)
```
问答 → 云端质检 → 知识提取 → 去重入库 → 触发微调 → 模型更新

氛围要点:
- 进化进度可视化（知识增长曲线）
- 贡献感（用户的提问帮助系统成长）
- 迭代成果可查看（本周新增XX条知识）
```

### 模块5: 实验控制面板 (Vibe: 严谨)
```
实验模式:
├─ baseline: 单模型无增强
├─ rag_only: 仅RAG增强
├─ expert_routing: 多专家路由
├─ full_system: 完整系统
└─ ablation_*: 消融实验组

氛围要点:
- 一键切换实验配置
- 实验数据自动采集
- 对比图表实时生成
```

## 🎨 用户体验地图

### 首次使用体验
```
1. 进入首页 → 看到清晰的学科选择
2. 输入问题 → 系统快速识别学科
3. 获取答案 → 专业、详细、有来源
4. 查看来源 → 知道答案从哪来
5. 持续使用 → 感受到系统越来越懂我
```

### 日常使用体验
```
1. 快捷提问 → 支持图片/文字
2. 多轮对话 → 上下文理解
3. 查看历史 → 复习之前的问答
4. 反馈纠错 → 帮助系统改进
5. 见证成长 → 看到知识库增长
```

### 实验测试体验
```
1. 选择实验模式 → 一键切换配置
2. 批量测试 → 导入基准数据集
3. 实时监控 → 看测试进度
4. 结果分析 → 自动生成对比图表
5. 数据导出 → 支持论文格式
```
