### 数据项2：各模块贡献度

#### 位置
- **章节**：5.3.3 - 5.3.6节
- **查找关键词**："17.5"、"9.8"、"3.5"

#### 计算方法
```
专家路由贡献 = expert_routing准确率 - baseline准确率

自我进化贡献 = full_system(3轮后) - full_system(初始)
```

#### 实验运行顺序
```
1. baseline实验（无专家路由，无RAG）
2. rag_only实验（无专家路由，有RAG）
3. expert_routing实验（有专家路由，有RAG，无迭代）
4. full_system实验（完整系统，记录初始和3轮迭代后）
```

## 实验运行步骤

### 步骤1：准备环境
```bash
# 1. 确保PostgreSQL运行
docker ps | findstr postgres

# 2. 启动后端
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# 3. 启动前端
cd frontend
npm run dev
```

### 步骤2：运行基准实验
```
1. 打开 http://localhost:3000
2. 进入"实验控制"页面
3. 点击"创建实验"
4. 配置：
   - 实验名称：baseline测试
   - 实验模式：baseline
   - 测试集：选择准备好的数据集
5. 点击"运行"
6. 等待完成，记录accuracy
```

### 步骤3：运行对比实验
```
重复步骤2，分别选择：
- rag_only
- expert_routing
- full_system

记录每个实验的accuracy和response_time
```

### 步骤4：计算贡献度
```
专家路由贡献 = expert_routing_acc - baseline_acc
RAG贡献 = rag_only_acc - baseline_acc
自我进化贡献 = full_system_3round - full_system_1round
```
