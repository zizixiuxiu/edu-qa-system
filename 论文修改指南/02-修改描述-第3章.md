3.1.3 系统使用场景设计

系统面向K12教育场景，支持以下使用模式：

系统采用匿名交互模式，无需注册登录即可使用，降低使用门槛。

## 修改项2：3.3.2节补充四级缓存

### 位置
- **章节**：第3章 → 3.3.2节末尾
- **行号**：3.3.2节最后1-2段之后

### 添加新小节
```
3.3.4 四级缓存架构设计

为降低响应延迟，系统实现四级缓存架构：
(1) L1内存缓存：进程级缓存，存储专家实例和向量结果，响应最快
(2) L2 Redis缓存：分布式缓存，支持跨进程共享（可选配置，未配置时自动降级）
(3) L3 PostgreSQL：持久化存储，保证数据可靠性
(4) 冷启动：按需加载专家模型，首次访问时初始化

该架构确保在高并发场景下仍能保持较低的响应延迟。
```

## 修改项3：3.4.1节补充纯图片输入

### 位置
- **章节**：第3章 → 3.4.1节
- **原内容**："支持图文混合输入"

### 修改为
```
3.4.1 多模态输入模式

系统支持三种输入模式：
(1) 纯文字输入：传统问答模式，用户直接输入问题文本
(2) 图文混合输入：用户上传图片并附带文字说明，VLM联合理解
(3) 纯图片输入：用户仅上传题目图片，系统自动使用"请解答图片中的题目"作为默认提示，VLM识别图片内容后路由到对应专家

对于纯图片输入，系统优先分析图片中的公式、图形、符号等内容，忽略用户可能附带的简短文字（如"解答这题"），确保学科识别准确性。
```

## 修改项5：3.6.5节改为数据导出

### 位置
- **章节**：第3章 → 3.6.5节
- **原小标题**：3.6.5 微调数据生成与模型迭代

### 修改为
```
3.6.5 知识导出与外部微调支持

系统不内置LoRA微调模块，而是通过数据导出支持外部开源工具进行微调。

数据导出流程：
(1) 系统自动将质检评分≥4.0的高质量问答对存入知识库
(2) 通过API接口/api/v1/knowledge/export导出数据
(3) 支持JSON/CSV/JSONL三种格式
(4) 可按学科、质量分数筛选导出内容

外部微调流程：
(1) 使用LLaMA-Factory、unsloth等开源工具加载导出数据
(2) 进行LoRA微调训练
(3) 导出Adapter权重
(4) 将微调后的模型加载回系统，更新专家配置

表3-X 导出数据字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| question | string | 问题文本 |
| answer | string | 答案文本 |
| subject | string | 学科分类 |
| quality_score | float | 质量评分(0-5) |
| source_type | string | 数据来源类型 |
| created_at | datetime | 创建时间 |
```
